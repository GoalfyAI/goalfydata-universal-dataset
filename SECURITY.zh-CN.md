# 安全策略

## 报告漏洞

如果你发现了安全漏洞，请以负责任的方式进行报告。

**请勿在 GitHub Issue 中公开报告安全漏洞。**

请发送邮件至：**goalfydata@goalfyai.com**

请包含以下信息：
- 漏洞描述
- 复现步骤
- 潜在影响
- 建议的修复方案（如有）

我们将在 48 小时内确认收到报告，并在 5 个工作日内提供详细回复。

## 受支持的版本

| 版本    | 是否支持 |
|---------|----------|
| v1.0.x  | 是       |

## 安全实践

- 所有凭证在静态存储时使用 AES-256 加密，传输中使用 TLS 1.3 加密
- 用户脚本在隔离沙箱中运行，无法访问宿主机
- API Token 可随时轮换或撤销
- 敏感数据不会记录在执行日志中
