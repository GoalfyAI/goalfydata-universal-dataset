# 核心概念

## GoalfyData 是什么？

GoalfyData 是一个用于 AI Agent 构建数据资产的运行时平台。Agent 负责创建资产，GoalfyData 负责让它持续运行。

## 三层架构：Build → Run → Share

### 1. Build — 让任何 Agent 都能构建数据资产

GoalfyData 不会替代你的 AI Agent，而是增强它。

通过 Skill / MCP / CLI，GoalfyData 融入你已有的 Agent 工作流：

- Claude Code
- Codex
- Cursor
- Manus
- ChatGPT / 自定义 GPTs

你的 Agent 可以自然生成：

- **数据集** — 从 API、文件或数据库结构化生成数据
- **更新脚本** — 用 Python / TypeScript 自动刷新数据
- **模板** — 可复用的数据结构与配置
- **数据面板** — Agent 构建的可视化数据应用

**关键洞察**：用户不需要先学习复杂平台，而是在自己已经使用的 Agent 里直接构建资产。

### 2. Run — 让资产持续运行与更新

这是 GoalfyData 与其他所有工具的核心区别。

Agent 输出通常是一次性产物：

- 只能在本地运行的脚本
- 一周后就过期的 CSV
- 关闭标签页就失效的 dashboard

GoalfyData 将它们转化为持续运行的服务：

- **定时刷新** — 基于 cron 的自动数据更新
- **沙箱执行** — 隔离环境运行更新脚本
- **版本控制** — 追踪变更，需要时可回滚
- **原子更新** — full_replace 模式确保不会出现空表中间态

Agent 可以构建一切，而 GoalfyData 负责让它持续运行。

### 3. Share — 安全、可控、跨 Agent 复用

当数据资产可以共享和复用时，它才真正产生价值。

GoalfyData 将分散文件转化为可管理、可共享的资产：

- **与同事共享** — 基于角色权限访问
- **与客户共享** — 只读或预览权限
- **团队共享** — 组织级数据资产
- **与其他 Agent 共享** — Agent-to-Agent 数据流

权限级别：查看 / 预览 / 查询 / 更新 / 管理

**解决的问题**：不再通过邮件发送 CSV，不再反复解释数据定义，也不再在每个新 Agent 对话中重新构建同一份数据集。

## GoalfyData 不是什么

理解边界和理解价值同样重要。

| GoalfyData 不是 | 它是 |
|---|---|
| AI Agent | Agent 构建的数据资产的运行平台 |
| BI 工具（Tableau/Looker） | Agent 构建数据面板的平台，不是人类手工构建 |
| 轻量版 dbt | Agent 生成的数据流水线的托管层 |
| MCP 工具/服务器 | MCP 连接数据之后的持久化层 |
| 通用应用托管（Vercel） | 专为 Agent 构建的数据资产设计的托管平台 |

GoalfyData 是所有 AI Agent 都需要的基础设施层，但任何单一 Agent 都不提供这一层能力。

## 为什么现在重要

三大趋势正在同时发生：

1. **Agent 已进入生产环境** — 57.3% 的组织已经在运行 Agent（LangChain Survey 2026）
2. **MCP 正成为标准** — 28% 的财富 500 企业已采用 MCP 作为基础设施
3. **输出持久化是缺口** — 所有人都在讨论 agent memory，但没有人解决 agent 输出的持续存在问题

GoalfyData 正是填补这一空白。
